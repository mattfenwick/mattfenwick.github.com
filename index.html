What have I done?
  • I currently work on the Connjur project, an open source effort dedicated to build software for NMR data analysis. Some of the problems I've worked on include: building a GUI for the Connjur Spectrum Translator project, implementing a tool for analyzing schemes for more efficiently collecting NMR data, building databases for storing and querying protein information, writing multiple parsers for converting between various NMR data formats, and solving protein structures using NMR.
  • I've written my own library for quickly building parsers. It works great, even for complicated languages!
  • I've used my parser library to build a parser for the NMR-Star data format, which is used by the BMRB to archive NMR data.
  • I defined and implemented my own programming language in order to learn about concrete syntax, abstract syntax, and formal semantics.
  • To hone my parsing skills and figure out some of the pitfalls, I built parsers for regular expressions, the Clojure language, the Java language, and the JSON format. Once you've built a parser for a language (not *using* it, but *for* it), you really realize how many dark corners there are lurking. For instance, can JSON objects have duplicate keys? :)
  • BioPython, nbubna/HTML, and hylang/hy are some of the projects I've contributed to.
A lot of my work has ended up in academic publications:
  • Parser Combinators: A Practical Application for Generating Parsers for NMR Data. Fenwick et al, ITNG 2013
  • Random phase detection in multidimensional NMR. Maciejewski et al, PNAS 2011.
  • An Open-Source Sandbox for Increasing the Accessibility of Functional Programming to the Bioinformatics and Scientific Communities. Fenwick et al, ITNG 2012.
  • Error Reporting Parsers: a Monad Transformer Approach. Fenwick & Vyas, The Monad Reader, issue 22, 2013.
